Detects duplicate parameters in Robot Framework keyword calls.

When a keyword is called with the same parameter specified multiple times, it often indicates a mistake. This inspection helps you identify and remove redundant parameter definitions to keep your test cases clean and maintainable.

Example:


My Keyword    arg1=value1    arg2=value2    arg1=value3

In this example, the parameter arg1 is defined twice with different values. This is likely unintended and should be corrected to use only one definition.

Corrected code:


My Keyword    arg1=value3    arg2=value2

The inspection will highlight all occurrences of duplicated parameters in keyword calls. Use the provided quick fix to navigate to and review the duplicate definitions.
