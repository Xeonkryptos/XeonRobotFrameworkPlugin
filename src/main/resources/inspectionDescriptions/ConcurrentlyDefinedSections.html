Reports concurrent definition of *** Tasks *** and *** Test Cases *** sections in the same Robot Framework file.

Robot Framework permits only one executable section per file: either tasks or test cases. Defining both makes the file invalid; tools may raise parse errors or ignore one section.

Why this matters: Mixing task and test semantics in a single file is ambiguous and not supported by the Robot Framework parser.

How to fix:

  • Keep only the section that matches the file's purpose.
  • Rename the header if you used the wrong one (e.g. change *** Tasks *** to *** Test Cases ***).
  • Split tasks and test cases into separate files.

Valid examples:

*** Tasks ***
Daily Cleanup
    Log    Cleaned

*** Test Cases ***
Login Succeeds
    Log    Logged in

Invalid (reported) example:

*** Tasks ***
Daily Cleanup
    Log    Cleaned

*** Test Cases ***
Login Succeeds
    Log    Logged in

Both section headers are highlighted as errors. Remove one to resolve the problem.
